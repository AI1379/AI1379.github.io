有关各种排序

本文最后更新于：5 个月前

本文比较散 适合有基础的同学 同时也给自己复习留下一点笔记

几个月不做算法题的后果就是发现自己连快排都不会打了

说是说关于各种排序，主要的篇幅应该还是会集中在大名鼎鼎的快速排序上

插入排序

先来插入排序主要是因为它在后面快排的优化里会提到。在冒泡、选择和插入这三种比较简单的算法里插入排序是效率最高的（但时间复杂度仍然是$O(n)$）。同时他也是很经典的一个算法，在《算法导论》最开始第一章（或许是第二章？这里记不太清了）讲算法分析便以之为例，这个之后有空再另写一篇。

插入排序简单而又直观。

想象一下洗牌的时候，随手抓了一把牌，我们得要把他们按照顺序排好。一般来说，我们会先在后面没有排过序的那些牌里选一张，然后在前面排过序了的牌里面找，找到应该要把这张牌插入在哪两个之间，把它插入进去。一般为了代码实现方便，都会取没排过序里最靠前的一张。

插入排序用的就是这个原理。直接贴代码吧。

void insertsort(int left, int right)
{
    int i, j, key;
    for (i = left + 1; i <= right; i++)
    {
        key = a[i];
        j = i - 1;
        while (j >= left && a[j] > key)
        {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key;
    }
    return;
}

其中a[]是要排序的数组。后面的代码也是一样，不再注明了。

快速排序

朴素算法

洛谷P1177 【模板】快速排序

注：这个题有毒瘤数据，朴素的排序算法只能过前三个点

快排是一个极其经典且优秀的算法，几乎每一种主流的语言都在自己的标准库里写了一套快排的某种实现（或许Python除外，它自带的sort()算法是另一套稳定的算法Timsort）

快排是一个典型的分治算法。它将整个序列以某个数为标准分出较小的和较大的两个部分，再将两个部分分别排序，最终使得整个序列被排序。下面是演示动画，来自WikiPedia。

算法步骤如下：

  1. 取基准值：从整个序列里任取一个数作为基准pivot，在这张图里用红色表示。这个基准的选取会直接的影响算法效率，如果这个基准没有取好可能会导致快排退化为冒泡排序。后面对于冒泡排序的一个优化就是改进基准值的选取。
  2. 分割：先把pivot移到最后面或最前面，再把比pivot小的放在前面，把比它大的放在后面，再把pivot放到两个序列中间。到这一步后，pivot这个数就已经排在它应该在的地方了，之后的过程不会再改变这个数的位置了。
  3. 递归：对前后两个没有排序的部分分别进行快排，直到序列里只有一个数。

朴素的快速排序算法平均时间复杂度是$O(n\log n)$，但最坏时间复杂度是$O(n^2)$，此时它退化成了冒泡排序。

优化算法

几种常见的优化思想如下：

  1. 优化pivot的选取，如三数取中或随机选取，可以避免一些极端数据（比如已经是升序或降序序列）带来的退化。
  2. 对于数据量较小的情况使用插入排序，这种优化同样适用于归并排序
  3. 每次排序时将与pivot相等的元素放在中间，以避免大量相同数据带来的退化。

下面是两个常见的优化

三数取中

实现了前面的第一点，在左、中、右三个数据中取中位数作为pivot，代码如下：

inline int choosePiovt(int left, int right)
{
    int x = left, y = right, z = (left + right) / 2;
    if ((y <= x && x <= z) || (z <= x && x <= y))
        return x;
    else if ((x <= y && y <= z) || (z <= y && y <= x))
        return y;
    else if ((x <= z && z <= y) || (y <= z && z <= x))
        return z;
}

三路快排

三路快排实现了前面的第三点。它与朴素的快排不同在于它将原序列分成了三部分：小于pivot，等于pivot和大于pivot三个部分，从而进行优化。
